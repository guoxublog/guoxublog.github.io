使用Hexo在GitHub上搭建博客

发表于 2021-09-09

前言

​ 一直想搭建自己的博客，发现可以在GitHub上用Hexo搭建自己的博客，非常方便，也很美观，就花了些时间建立了自己的博客，在此把搭建过程记录下来，方便其他人参考。

准备工作

  • 建立 GitHub 账号
  • 下载 node.js
  • 下载 Git for windows

本文所用环境

  • Windows 10
  • node.js 14.17.6
  • Git 2.33.0
  • hexo 2.0.0

GitHub搭建

​ 首先得建立的一个仓库来存储博客，名为 你的用户名.github.io 。比如我的GitHub账号为guoxublog，则仓库名称为pandablog.github.io，如下图所示：

本地环境搭建

安装node.js 和 Git for Windows

​ 按照默认安装即可。安装完成之后，打开命令窗口（Win键+R，输入cmd，回车），分别输入node -v，npm -v，git –version，如果像下图一样显示版本号说明安装成功。

SSH授权

​ 为什么要配置这个呢？因为你提交代码肯定要拥有你的github权限才可以，但是直接使用用户名和密码太不安全了，所以我们使用ssh key来解决本地和服务器的连接问题。

​ 打开git bash，输入ssh-keygen -t rsa, 接着回车三下，这样就会在C盘用户目录下生成id_rsa和id_rsa.pub这两个文件，前者是密钥，后者是公钥，用记事本打开id_rsa.pub，复制其中的全部内容，添加到GitHub上，这样本地的id_rsa密钥就可以和GitHub上的id_rsa.pub公钥进行配对，授权成功。

安装Hexo

​ 在任意一个地方新建一个文件夹，在git bash中进入这个文件夹，比如**cd /e/Homepage/hexo/**（之后安装的内容会保存在这个文件夹内），输入npm install -g hexo安装hexo即可。

​ 然后输入npm install hexo –save这个时候你会看到命令行出现了一堆白字，紧接着输入hexo v查看是否安装成功（这里我安装了next主题，所以会显示next大字，后面会介绍next主题）

初始化hexo

​ 继续输入hexo init实现初始化（注意：在原文件夹下初始化可能失败，需要在其他文件夹下初始化，然后把生成文件复制到之前文件夹，比如 ”e/Homepage/hexo“）

​ 然后输入hexo s

​ 这时候我们就可以打开浏览器了，在地址栏中输入http://localhost:400/，我们的博客就建立成功啦！

​ 我们可以去看一下我们的源码的位置，就在之前新建的那个文件夹里，在source->_posts文件夹下，有一篇helloworld的初始化文章，如果想要添加新的文章，可以在命令行输入hexo new ‘filename’,就会生成一个新的.md文件，对它编辑就可以了。

​ 部署好之后，我们就可以上传我们的代码了

　　回到命令行窗口，输入npm install hexo-deployer-git –save,

　　再输入hexo g，然后再hexo d，就可以将我们public里面的代码上传，在GitHub上可以看到我们上传的代码。这样别人也可以通过域名访问我们博客了。在地址栏输入http://域名就可以访问。

更改主题
